#  WinCmdRun-GUI 渗透工具箱说明文档
# 项目概述
WinCmdRun-GUI 是一款基于 JavaFX 开发的渗透测试辅助工具，旨在通过图形化界面便捷地管理和执行各类渗透测试程序，支持批量执行、参数配置及日志记录等功能。


# 注意事项

```
* 配置文件说明：所有配置信息均存储在程序根目录的 config.json 文件中。若删除程序但保留该文件，重新下载程序后可直接读取原有配置；首次启动程序时，会自动生成 config.json。
* 配置保存规则：
    1、程序分组、程序列表新增后会自动保存至配置文件。
    2、其他配置（如程序路径、参数等）需满足：先选中程序列表中的程序，且程序名称不为空，配置完成后必须点击 “保存配置”，否则配置会丢失。
* 文件清理说明：根目录中除 WinCmdRun.jar（主程序）和 config.json（配置文件）外，其他自动生成的文件可自由删除，不影响程序运行。
```
# 程序核心目标

```
1、快速启动：支持一键启动 exe、py、jar 和其他可以双击打开的类型的文件。
2、灵活配置启动命令：可快速配置带参数的启动命令（适用于 .py    .exe    .jar 等需要参数的文件）。
* 目前支持两个可变参数：[ip]（IP 地址）和 [port]（端口）。
* 参数输入方式：在 “目标” 输入框中填写，用 @ 分隔 ip 和 port（例如：127.0.0.1@3389）。
* 示例：
      命令模板：[file] -h [ip] → 实际执行可以是：程序路径 -h 127.0.0.1
      命令模板：[file] -h [ip] → 实际执行可以是：程序路径 -h http://127.0.0.1:8090
      命令模板：[file] -h [ip] -p [port] → 实际执行可以是：程序路径 -h 127.0.0.1 -p 3389
```

# 使用方法：


配置本地地址和启动命令后，点击保存按钮，保存完成后，在目录栏目中输入IP或者相关的参数点击启动程序

# 核心功能
* 程序分组管理：支持创建、删除、重命名分组，便于对渗透工具进行分类管理
* 程序配置管理：可添加、编辑、删除程序信息，包括本地路径、远程更新路径等
* 批量执行功能：支持基于目标列表批量执行程序，配合参数变量实现灵活调用
* 参数模板管理：提供 3 组参数配置，支持变量替换和日志记录
* 日志记录：可将程序执行结果保存到日志文件，便于后续分析


# 技术架构
* UI 框架：JavaFX 用于构建图形用户界面
* 数据存储：使用 JSON 格式保存配置信息（基于 Gson 库）
  

# 使用流程
1. 分组管理
* 右键点击左侧分组列表，可执行 "新增分组"、"删除分组"、"重命名分组" 操作
* 点击分组名称可切换当前分组，右侧程序列表会同步更新
2. 程序管理
* 在选中分组后，右键点击右侧程序列表，可执行 "新增程序"、"删除程序"、"重命名程序" 和 "移动到分组" 操作
* 选中程序后，下方表单会显示该程序的详细配置，修改后点击 "保存配置" 生效
3. 参数配置
* 支持 3 组启动参数（Param1/Param2/Param3），通过单选按钮切换
* 参数支持变量替换：
* [file]：程序文件的绝对路径
* [ip]：目标文本中的 IP 地址（需目标格式包含 IP）
* [port]：目标文本中的端口号
* [log]：添加此参数可将执行结果保存到日志
4. 目标配置
* "目标" 文本框支持多行输入，每行代表一个执行目标
* 执行程序时会按顺序对每个目标执行一次程序
* 目标为空时，程序会执行一次默认命令
5. 功能按钮
* 浏览本地路径：选择程序的本地可执行文件
* 浏览更新路径：通过浏览器打开远程更新链接
* 启动程序：按当前配置执行程序（支持批量执行）
* 保存配置：保存当前程序的所有参数设置
* 打开文件夹：打开程序本地路径所在的目录
* 使用说明：显示详细使用指南

  
# 配置存储
* 程序配置保存在 config.json 文件中
* 分组信息和程序信息会自动持久化
* 执行日志（当使用 [log] 参数时）保存在根目录的 log.txt 文件中
